# Security Policy

## Supported versions

Only the latest 0.1.x release line is considered supported during the MVP phase.

## Reporting a vulnerability

Do not open a public GitHub issue for security-sensitive reports.

Send the report privately to the maintainer before disclosure. Include:

- affected version
- reproduction steps
- impact summary
- logs or screenshots if available

Until a dedicated security mailbox exists, use the repository contact route and mark the report as security-sensitive.

## Scope notes

- Issues limited to local development signing or expired iOS developer trust are not treated as security vulnerabilities.
- Bridge auth token leakage, unauthorized device access, or unintended transcript exposure are in scope.
